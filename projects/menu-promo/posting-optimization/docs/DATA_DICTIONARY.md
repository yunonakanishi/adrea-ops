# DATA_DICTIONARY.md — データソース定義

## 分析対象
- **対象エリア**: menu_database_area.csv に記載の515エリアのみ
- **対象期間**: 2024年1月〜2026年3月（約25ヶ月）

## 6テーブルの定義

### ① menu_database_area.csv（515行・206列）
エリアの人口動態・世帯構成・年収分布などの属性データ。

| 項目 | 内容 |
|------|------|
| 結合キー | `エリア` 列 |
| 主要カラム | 人口、世帯数、年齢別人口比率（5歳刻み）、1人世帯比率、年収帯別世帯数、持ち家/借家比率、緯度経度 |
| 用途 | エリア属性とCVRの相関分析 |

### ② menu_result_order_area.csv（1,569行）
エリア別の注文実績。ポスティング起因に限らない全注文の集計。

| 項目 | 内容 |
|------|------|
| 結合キー | `エリア名` 列 |
| 主要カラム | 平均単価（クーポン込み/抜き）、クーポン利用率、1ユーザーあたり注文回数、平均LTV（クーポン込み/抜き） |
| 用途 | エリアのLTV傾向（参考値。ポスティング起因かどうかは区別不可） |

### ③ menu_database_area_posting.csv（1,372行）
ポスティング配布可能エリアのマスタ。

| 項目 | 内容 |
|------|------|
| 結合キー | `エリア名` 列 |
| 主要カラム | 最大配布可能部数（JBF）、配布単価、配布費、配布期間、h3index大、緯度経度、都道府県、市区町村 |
| 用途 | CPA算出時のコスト情報、配布期間→月間配布回数の算出 |

配布期間→月間配布可能回数の変換:
- 「1週間」→ 月4回
- 「2週間」→ 月2回
- 「3週間」「1ヶ月」「1カ月」「1ヵ月」→ 月1回
- それ以外 → 月1回（安全側）

### ④ menu_result_posting_area.csv（13,299行）
施策×エリア別のポスティング実績。**注文地点ベース**のCV数。

| 項目 | 内容 |
|------|------|
| 結合キー | `施策名` + `エリア名` |
| 主要カラム | 配布部数、総費用（税抜）、印刷費（税抜）、配布費（税抜）、配布開始日、初回購入人数（配達完了のみ）、初回購入人数（キャンセル含む）、グロス費用、CPA |
| CVR算出 | 初回購入人数（配達完了のみ）÷ 配布部数 |
| CPA | CSVに入っているCPA列をそのまま使用（= グロス費用 ÷ 初回購入人数） |
| 用途 | エリア評価、時期評価 |

### ⑤ menu_result_posting_couponcode.csv（10,590行）
施策×クーポンコード別の実績。**クーポンコードベース**のCV数。デザイン情報を持つ。

| 項目 | 内容 |
|------|------|
| 結合キー | `施策名` + `クーポン` |
| 主要カラム | 配布部数、総費用、デザイン、クーポン、合計注文件数、初回購入人数（配達完了のみ）、2回目購入人数、合計利用クーポン額、合計購入単価、初回購入人数（キャンセル含む）、グロス費用、CPA |
| 用途 | デザイン評価 |

### ⑥ menu_result_posting_couponcode_area.csv（41,028行）
クーポンコード×エリア別のCV数。⑤を注文地点で分解したもの。

| 項目 | 内容 |
|------|------|
| 結合キー | `クーポンコード` + `エリア名` |
| 主要カラム | 初回購入人数（配達完了のみ）、初回購入人数（キャンセル含む） |
| 用途 | デザイン×エリアのクロス分析（ABテスト検証） |

**注意**: ⑥のクーポンコード列には `DELI-` プレフィックスが付いている。⑤のクーポン列にはプレフィックスがない。結合時はプレフィックスを除去して突合する。

## テーブル間の結合関係

- ④と⑤は行単位では結合しない。同一エリアに複数デザインをランダム配布するABテストが存在するため粒度が異なる
- ⑥が④と⑤のブリッジとして機能する
- ①の`エリア`列と他テーブルの`エリア名`列で結合（表記は同じだがカラム名が異なる）

## CVの定義差異（許容済み）

| テーブル | CV数の基準 | 用途 |
|---------|----------|------|
| ④ posting_result_area | 注文地点ベース | エリア評価（統一基準） |
| ⑤ posting_result_couponcode | クーポンコードベース | デザイン評価 |
| ⑥ couponcode_area | コード×注文地点 | デザイン×エリアのクロス |

④と⑤⑥で同一エリアのCV数が一致しないケースがある。配布地点≠注文地点のケースに起因し、許容する。

## CSVの注意事項

- エンコーディング: UTF-8 BOM付き（`utf-8-sig`で読む）
- 数値カラムにカンマ区切りあり（`"63,500"` など）→ `str.replace(',', '')` してからint/float変換
- 欠損値・空欄のレコードは集計から除外（0と欠損の区別がつかないため）
- 配布部数1,000部未満のレコードは集計から除外（CVRのブレが大きくノイズになるため）
- CPA列はCSVに入っている値をそのまま使用。自前で計算しない（グロス費用ベース）
