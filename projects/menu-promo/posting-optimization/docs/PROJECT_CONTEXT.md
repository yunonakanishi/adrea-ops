# PROJECT_CONTEXT.md — プロジェクトの背景と経緯

## プロジェクト概要

menuアプリのポスティング（紙チラシ配布）施策を「慣性投資」から「データとルールに基づく最適化投資」へ変革するプロジェクト。

- **担当**: アドレア 中西
- **期間**: 2026年3月末まで（クーポン検証のみ5月末）
- **クライアント**: menu社 責任者陣

## 4つの取組み領域

1. **計測基盤の構築**: KPI（CVR・CPA・LTV）の可視化とトラッキング
2. **情報収集**: エリアデータ・競合ポスティングデータの資産化
3. **自走運用の設計**: 業務フロー刷新・マニュアル化・予算配分ロジック
4. **CPA最適化プロセスの確立**: コスト削減とCVR改善のPDCA

## 重要な意思決定の履歴

### 1. クーポンコード統一の判断

**背景**: 従来はエリアごとにクーポンコードを発行していた（例: ABC123 = 四谷エリア専用）。今後はコストの削減のためコードを施策ごとに統一する方針。

**問題**: コード統一すると「どのコード入力がどの配布エリア由来か」がわからなくなる。

**判断**: コード統一してよい。理由:
- 旧方式（コードベース）も正確ではなかった（四谷で配ったチラシを秋葉原で使うケースがある）
- 今後は注文地点ベースでCVRを集計する
- 過去分も注文地点ベースで再集計し、同じ物差しで比較できるようにする

**CVRへの影響**:
- デザイン評価にはノイズが入らない（コード入力＝CV発生の事実は変わらない）
- エリア評価のみ精度が下がる（配布地点と注文地点のズレ）

### 2. CVR計測基準の統一

- **④ posting_result_area**: 注文地点ベース → 今後の統一基準
- **⑤ posting_result_couponcode**: クーポンコードベース → 旧基準（参考値として保持）
- **⑥ couponcode_area**: コード×注文地点のクロス → ブリッジテーブル
- ④と⑤⑥で同一エリアのCV数が一致しないケースがある → **許容する**

### 3. CPA定義の変更

途中でCPAの定義が変わった。

- **旧定義**: CPA = 総費用（税抜）÷ 初回購入人数（配達完了のみ）
- **新定義**: CPA = グロス費用 ÷ 初回購入人数（配達完了のみ）
- グロス費用 = （配布費 + 印刷費）× 1.25（代理店マージン25%込み）
- ④⑤のCSVに「グロス費用」「CPA」カラムが追加された
- 全分析結果をグロス費用ベースで再算出済み

### 4. エリア属性分析の結論

**分析の過程**:
1. 単純相関（デザイン混在）→ r_max = 0.134。「エリア属性は効かない」と見えた
2. デザイン固定×CV数（実数）→ r = 0.4〜0.7の強い相関。しかし人口規模の交絡だった
3. デザイン固定×CVR（率）→ r_max = 0.15〜0.28。人口交絡を除去すると効果は限定的
4. 施策制御偏相関・重回帰 → r_max = 0.139、有意変数が49/80に急増。ΔR²=8.2%

**結論**: 「エリア属性は効かない」ではなく「効くが影響は小さい」。施策を制御した分析で初めて検出できた。

**CVRを高めるエリア属性**:
- 25-29歳比率 (r = +0.133)
- 30-34歳比率 (r = +0.126)
- 労働人口比率 (r = +0.107)
- 1人世帯比率 (r = +0.104)

**CVRを下げるエリア属性**:
- 55-59歳比率 (r = -0.139)
- 60-64歳比率 (r = -0.138)
- 65歳以上世帯員 (r = -0.113)

### 5. 予算配分ロジックの設計

**配分ロジックの仕様（最新）**:

インプット:
- 許容CPA（円）
- 予算（円・グロス費用ベース）
- 配布月（1-12）
- エリア指定（任意。指定があれば必ず含める）
- 印刷単価（任意。省略時は過去実績の中央値）
- モード: "cpa"（許容CPA厳守）/ "budget"（予算消化優先）/ 省略時は両方出力

ロジック:
- エリアごとの配布期間から月間配布可能回数を算出（1週間→月4回、2週間→月2回、3週間以上→月1回）
- 月間最大配布部数 = JBF × 月間配布可能回数
- 期待CPA低い順にエリアを追加
- cpaモード: 全体の加重平均CPAが許容CPA以下の範囲で選定
- budgetモード: 予算が尽きるまで選定
- 指定エリアはCPAに関係なく必ず含める

アウトプット:
- エリアごとの配布部数・配布期間・配布費用・想定CVR・配布回数・ランク
- 全体の想定印刷費用・想定合計費用・想定CV数・想定CPA

**注意**: スクリプト（allocation_calculator.py）は再設計中。上記仕様でのコーディングはこれから。

### 6. デザイン分析の限界

過去データから分かったこと:
- 訴求タイプでCPAに最大7.5倍の差（金券系4,594円 vs クーポン金額系34,262円）
- スクラッチ・くじ系はCVR最高（0.200%）だが印刷コスト高でCPAでは金券系（4,594円）に劣後
- ABテスト勝敗データは6,715ペアあるが、CV数ベースの比較でCVR率での比較ではない

過去データでは分からないこと:
- デザインの「何が効いているか」の言語化 → デザイナーとの手動分類が必要
- ABテストの配布設計が不統一（エリア内ランダム配布とエリア別配布が混在）

今後の方針:
- CR検証フレームワーク（ABテストルール・PDCA）を策定中（3月中完了予定）
- 1施策1変数テスト、同一エリア内ランダム配布を原則化
- 0.05%pt差の検出に1群124,249部が必要
