要件 受領・確認 アドレア

X-60日
担当アドレア（浅葉・川野）
目的
menu社から当月施策の前提条件（予算・エリア・許容CPA等）を受領し、ポスティングの要件を確定させる。
INPUT
  • menu社からのSlack連絡
OUTPUT
  • 整理済の要件（予算・エリア・許容CPA・獲得件数）
  • 不明点があれば確認依頼
手順
1
menu_user_marketing でmenu社からの施策要件共有を受領する
2
受領内容を確認し、ポスティングに関する要件（エリア・予算・許容CPA・検証内容）を整理する
3
不明点や不足情報があれば、同チャンネルでmenu社に確認する
注意点
  • 本来はここでポスティングの要件を握っておくべきだが、現状は都度Slackで確認している。受領時に要件を明示的にまとめておくこと
  • 実態はX-45日前に来ることが多い。遅延した場合は以降の全工程が圧縮されるため、早めの催促が重要

予算プランニング＋エリアプランニング アドレア

X-50日
担当アドレア（浅葉・川野）
目的
月のプロモ全体予算から各施策の予算配分と初回購入数獲得SIMを作成する。同時にエリアプランニングも実施する。
INPUT
  • 受領した施策要件
  • 前月CPA実績
  • 前月クリエイティブ効果
OUTPUT
  • BTLプランニングシート（予算配分・SIM記載済）
  • プロモーション承認依頼シート
  • プラン作成方針テキスト
手順
1
アドレア各部門の担当者が、各自でCPAとCV数を算出する
2
取りまとめ担当がmenu要望に合うように BTLプランニングシート を整える
3
エリアプランニング：前月配布の当月CPAとCRを見て配布エリアを決定する
要確認 エリア選定の具体的な判断基準・閾値ルール
注意点
  • 複数SIM作成依頼でシートが爆増することが多い。バージョン管理に注意
  • BTLプランニングシートと承認依頼シートが二重管理になっている
  • エリアプランニングは担当者の匙加減に依存しがち

入稿データ作成 アドレア JBF/アト確認

X-35日
担当アドレア（川野）
目的
予算に合わせて配布エリアを選定し、デザインとクーポンコードを割り当てた入稿データを完成させる。
INPUT
  • エリアプランニングシート
  • 確認済デザインデータ
  • クーポンコード一覧
  • 確定予算
OUTPUT
  • ポスティング発注管理（エリア×デザイン×クーポン割当完了）
  • mihon.pdf
手順
1
エリアプランニングシート でCPA平均順位・ランクを参考にエリア選定
要確認 エリア選定の判断基準・閾値ルール
2
デザイン分配：主力中心、検証用は1種20〜30万部目安
3
クーポンコードを各エリア×デザインの組み合わせに割当
4
ポスティング発注管理 のB〜H列を入力
5
menu_posting_jbf でJBF/アトに確認依頼：E列・H列の確認＋印刷単価記入を依頼
6
JBF/アト修正返却後：〇×チェック・部数修正・デザイン名と入稿データを確認
7
ダブルチェック（→「入稿データ確認」）実施後、menu_posting_jbf でGO連絡
注意点
  • コード重複がないか必ず確認（過去ミスあり）
  • 行ズレに注意 — エリア行とデザイン・クーポン行のずれは全配布に影響
  • mihon.pdfの記載情報と実際の割当の一致を目視確認

効果計測 アドレア

毎日
担当アドレア（川野）
目的
担当施策のデイリー効果を確認し、月次効果表の着予を最新化する。計画比で大きな乖離があれば原因を特定する。
INPUT
  • デイリー実績（Salesforce orgレポート）
  • 月次効果表【効果表OFF】
  • 月次効果表【効果表_OFF_後追い】
OUTPUT
  • 着予更新済の月次効果表
  • 好調/不調時：原因メモ
手順
1

2

4
好調/不調時：媒体担当者に配布完了率と要因を確認し、コメント欄に記入
要確認 好調/不調の判定閾値
頻度と更新タイミング
確認は毎日。着予の本格更新は毎週月曜。月曜の更新結果は同日17:30〆の「事業運営レポート提出」にそのまま使う。
注意点
  • 配布開始直後はCV数のブレが大きい。安定するまで1週間程度見ること
  • 後追いSIMの更新を忘れがち
